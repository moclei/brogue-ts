# Audit: Math.c

**Status:** Complete
**Audited:** 2026-03-06
**Auditor note:** c-inventory.md captured only 5 of 15 functions (static-only capture). Supplemented
with the public-function grep pass (TASKS.md workaround). Total: 10 public + 5 static = 15 unique
functions. `rand_range` appears twice in the grep output (lines 141 and 160) due to conditional
compilation (`#ifdef`-style variant); both map to the single `randRange` TS function and are counted
as one entry.

Math.c is the RNG and fixed-point arithmetic module: Bob Jenkins' small PRNG (ranval/raninit/range),
public RNG API (rand_range, rand_64bits, rand_percent, randClump, randClumpedRange, shuffleList,
fillSequentialList, seedRandomGenerator), and fixed-point math (fp_round, fp_sqrt, fp_pow, with
internal helpers msbpos and fp_exp2).

In the TS port, Math.c responsibilities are split across two files:
- `rogue-ts/src/math/rng.ts` — RNG primitives and public RNG API
- `rogue-ts/src/math/fixpt.ts` — fixed-point arithmetic

Both files are fully ported with real implementations. Every exported function has direct passing
tests in `rng.test.ts` and `fixpt.test.ts`, with cross-validation against known C output (reference
values generated by compiling and running the C source). Private/static functions are tested
transitively through their public callers.

**Math.c is 100% IMPLEMENTED — the only file in the audit with zero gaps.**

## Notes for follow-on initiative

**Math.c requires no fix work.** Both `rng.ts` and `fixpt.ts` are complete, well-annotated ports
with direct passing tests cross-validated against C reference output.

**Test quality is the highest in the audit.** rng.test.ts includes determinism verification,
stream-isolation checks, and statistical distribution checks for randPercent. fixpt.test.ts
includes exact-match cross-validation against C lookup table values (e.g., SQUARE_ROOTS[4] = 131073).

**Conditional compilation note:** `rand_range` has two C definitions (lines 141 and 160), likely
selected by a BROGUE_LIBTCOD or platform preprocessor guard. The TS port collapses both variants
into `randRange`. No action needed — the TS behavior matches the standard (non-TCOD) C path.
