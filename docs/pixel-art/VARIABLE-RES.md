# Variable-resolution sprites — implementation summary

This document describes the final state of the variable-res sprite implementation. It supersedes the earlier design note in this file.

---

## Overview

The variable-res feature allows individual `DisplayGlyph` entries (e.g. monsters, the player) to be sourced from larger sprite regions (e.g. 32×32 or 64×64 px) while still drawing into a single dungeon cell on screen. Source sheets declare a per-sheet `stride` in `tileset-manifest.json`; the sprite assigner stores tile-count dimensions (`w`/`h`) per glyph assignment and exports per-stride atlases during generation. At runtime, `SpriteRef` carries optional `srcW`/`srcH` pixel dimensions, `buildGlyphSpriteMap` reads them from `sprite-manifest.json`, and `SpriteRenderer` uses them for source-rect sampling, tinting, and bitmap precreation. All existing 16×16 manifests and assignments parse correctly without modification.

---

## Data shapes (final)

### `tileset-manifest.json` — per-sheet stride

Each `SheetDef` in the `sheets` array of a tileset entry may carry an optional `stride` field:

```jsonc
{
  "tileSize": 16,
  "basePath": "...",
  "tilesets": [
    {
      "id": "default",
      "name": "Default",
      "sheets": [
        { "key": "master",    "path": "master-spritesheet.png",  "category": "tiles" },
        { "key": "creatures", "path": "creatures-hires.png",     "category": "glyphs", "stride": 32 }
      ]
    }
  ]
}
```

`stride` is the tile stride in pixels for that sheet. When absent, the manifest-level `tileSize` (16) is used.

### `assignments.json` — glyph entries and sheets record

Glyph and tiletype entries use grid coordinates (`x`, `y`) plus optional tile-count dimensions (`w`, `h`). The top-level `sheets` record maps sheet keys to filenames:

```jsonc
{
  "sheets": {
    "master":           "master-spritesheet.png",
    "master-32":        "master-32.png"
  },
  "tiletype": {
    "FLOOR": { "sheet": "master", "x": 0, "y": 0 }
  },
  "glyph": {
    "G_RAT":    { "sheet": "master",   "x": 3, "y": 1 },
    "G_DRAGON": { "sheet": "creatures", "x": 0, "y": 0, "w": 2, "h": 2 }
  },
  "autotile": { ... }
}
```

- `sheet` — key into the `sheets` record; identifies which source PNG the coordinates refer to.
- `x`, `y` — grid position in tile units (multiplied by the sheet's stride to get pixel coordinates).
- `w`, `h` — region size in tile counts; both default to `1` when absent.

### `sprite-manifest.json` — glyph entries

Generated by the assigner. Glyph entries mirror the output atlas layout:

```jsonc
{
  "tiles":  { "FLOOR": { "x": 0, "y": 0 } },
  "glyphs": {
    "G_RAT":    { "x": 3, "y": 1 },
    "G_DRAGON": { "x": 0, "y": 0, "w": 64, "h": 64, "sheet": "master-32" }
  }
}
```

- `x`, `y` — grid column/row in the output atlas.
- `w`, `h` — pixel dimensions of the sprite in the output atlas; omitted for standard 16×16 entries.
- `sheet` — output atlas key; omitted when the entry is on the default `"master"` atlas.

### `SpriteRef` interface

```typescript
export interface SpriteRef {
  sheetKey: string;
  tileX: number;
  tileY: number;
  srcW?: number;   // pixel width; defaults to TILE_SIZE (16) when absent
  srcH?: number;   // pixel height; defaults to TILE_SIZE (16) when absent
}
```

`srcW`/`srcH` are in pixels and are used directly as the source-rect dimensions in `drawImage` and `createImageBitmap` calls.

## HMR path

No changes were required. `buildSheetUrls` already iterates all keys in `assignments.sheets`, `loadTilesetImages` accepts a `Record<string, string>` URL map and loads every entry, and the bootstrap `tileset-update` handler rebuilds all maps from the freshly fetched `assignments.json` and `sprite-manifest.json`. Adding new atlases (e.g. `master-32.png`) is picked up automatically on the next save-triggered HMR cycle.

---

## Backward compatibility

Existing `assignments.json` files without a `sheets` record fall back to a hardcoded `master-spritesheet.png` URL in `buildSheetUrls`. Glyph entries without `w`/`h`/`sheet` fields are handled at two levels:

- **Assigner** (`migrateAssignments`): normalises loaded data to `w: 1, h: 1` on every `SpriteRef`.
- **Runtime** (`buildGlyphSpriteMap`): defaults `srcW`/`srcH` to `TILE_SIZE` (16) and `sheetKey` to `"master"` when those fields are absent from `sprite-manifest.json`.

No existing sprite-manifest or assignments file requires manual migration.
