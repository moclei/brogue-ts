# Pixel Art Variable Resolution — Plan

## Approach

A four-layer pipeline, top to bottom:

```
tileset-manifest.json (per-sheet stride)
        ↓
assignments.json (glyph entries with w/h in tile units, hires sheet keys)
        ↓
sprite-manifest.json (glyph entries with w/h in pixels, sheet key)
        ↓
SpriteRef (srcW/srcH in pixels) → SpriteRenderer draw call
```

The assigner tool owns the top two layers (registry + source refs). The bottom
two layers are game-side (generated manifest + runtime lookup). Data flows
downward; the game never reads `tileset-manifest.json`.

Work is sequenced so each phase can be verified before the next begins:

1. Type contract (both sides) — no behavior changes, just types + tests
2. Game renderer — use per-ref dimensions; tint canvas, bitmap cache
3. Assigner backend — generate per-stride atlases, write updated `assignments.json`
4. Assigner UI — stride-aware grid, rect selection, hires thumbnails
5. Integration — verify full pipeline end-to-end; HMR smoke test
6. Manual testing — user-driven sign-off

## Technical Notes

### Data shapes

**`tileset-manifest.json` — assigner sheet registry**

Per-sheet `stride` field (optional, defaults to manifest-level `tileSize: 16`):

```json
{ "key": "MyBigSheet", "path": "...", "category": "creature", "stride": 64 }
```

Drives `GridCanvas` grid line interval and tile-coordinate-to-pixel math.

**Glyph assignment in assigner state / `assignments.json`**

Extended from `{ sheet, x, y }` to `{ sheet, x, y, w?, h? }` where `w`/`h`
are tile counts relative to the source sheet's stride. Default: 1×1.

Output pixel size = `w * stride` × `h * stride`.

```json
"G_RAT": { "sheet": "MyBigSheet", "x": 0, "y": 0 }              // stride=64 → 64×64
"G_GOBLIN": { "sheet": "DawnLike", "x": 3, "y": 2, "w": 2, "h": 2 } // stride=16 → 32×32
```

**`sprite-manifest.json` — generated by assigner**

Glyph entries extended from `{ x, y }` to `{ x, y, w?, h?, sheet? }` where
`w`/`h` are pixels and `sheet` is the output atlas key:

```json
"G_RAT":    { "x": 0, "y": 0, "w": 64, "h": 64, "sheet": "master-64" }
"G_GOBLIN": { "x": 1, "y": 0, "w": 32, "h": 32, "sheet": "master-32" }
"G_JACKAL": { "x": 5, "y": 2 }   // no w/h/sheet → defaults to 16px master
```

`x`/`y` are grid indices in the output atlas at the sprite's native stride.

**`assignments.json` — generated by assigner**

`sheets` key extended from `{ master: string }` to `Record<string, string>`:

```json
"sheets": {
  "master": "master-spritesheet.png",
  "master-32": "master-32.png",
  "master-64": "master-64.png"
}
```

**`SpriteRef` — game runtime**

```ts
interface SpriteRef {
  sheetKey: string;
  tileX: number;
  tileY: number;
  srcW?: number;   // pixel width, default TILE_SIZE
  srcH?: number;   // pixel height, default TILE_SIZE
}
```

`buildGlyphSpriteMap` reads `w`/`h`/`sheet` from manifest and populates
`srcW`, `srcH`, `sheetKey`. Standard glyphs without these fields get defaults.

`buildSheetUrls` iterates all entries in `assignments.sheets` (not just `master`),
so all output atlases are loaded automatically.

### Renderer changes

**`MAX_SPRITE_SIZE = 64`** constant added to `tileset-loader.ts`.

**`tintCanvas`** created at `MAX_SPRITE_SIZE × MAX_SPRITE_SIZE` in the
`SpriteRenderer` constructor. All `clearRect` / `drawImage` / `fillRect` calls
inside `drawSpriteTinted` switch from hardcoded `TILE_SIZE` to
`srcW = ref.srcW ?? TILE_SIZE`, `srcH = ref.srcH ?? TILE_SIZE`.

**`precreateBitmaps`** passes `srcW`/`srcH` to `createImageBitmap` for the
source crop. `bitmapKey` includes dimensions when they differ from `TILE_SIZE`:
`"sheetKey:tileX:tileY"` for standard; `"sheetKey:tileX:tileY:WxH"` for hires
(avoids future collision if indices are reused across atlases at different strides).

ENTITY layer has `blendMode: "none"` so tinting is skipped for creature sprites.
The tint canvas resize is primarily defensive (for F2 overrides and future STATUS
layer).

### Assigner output atlases

`generate.ts` groups glyph assignments by output pixel size. For each group:

1. Compute a layout: row-major grid, width fixed at `MASTER_GRID_W` columns
   or a sensible square, height expanded to fit.
2. For each glyph in group, extract the source rect `(x*sourceStride, y*sourceStride,
   w*sourceStride, h*sourceStride)` from the source sheet image.
3. Composite into a `sharp`-based output buffer at `outputStride` per cell.
4. Write to `rogue-ts/assets/tilesets/master-{outputStride}.png`.
5. Write manifest entry `{ x: col, y: row, w: outputStride, h: outputStride,
   sheet: "master-{outputStride}" }`.

Standard 16px glyphs use the existing master sheet path unchanged.

### Assigner UI — GridCanvas

`GridCanvas` looks up the active sheet's `stride` from the sheet registry
(loaded from `tileset-manifest.json`). Grid lines are drawn at `stride`-px
intervals instead of hardcoded 16.

Mouse coordinates convert to tile: `tileX = floor(mouseX / (stride * zoom))`.

For DisplayGlyph mode, click-drag is tracked to produce a rect selection
`{ tileX, tileY, w, h }`. For TileType mode, selection remains 1×1 (terrain
stays 16×16). Selection dimensions shown in `SelectionBar` as
`"2×2 tiles (32 px)"`.

### HMR path

No structural changes needed. Verified in Phase 5:

- `buildSheetUrls` iterates `Object.entries(assignments.sheets)` (with legacy
  fallback when `sheets` is empty), so all named atlas keys are returned.
- `loadTilesetImages` accepts `Record<string, string>` and loads every entry;
  no single-URL or fixed-shape constraint.
- The `tileset-update` HMR handler in `bootstrap.ts` calls `fetchAssignments()`,
  then `buildSheetUrls(newAssignments, ...)`, then `loadTilesetImages(newUrls)` —
  no hardcoded `master` key, all sheets reloaded on save.

No gaps found; no patches applied.

## Rejected Approaches

- Multi-cell packing in master sheet: layout math complexity and collision risk
  outweigh simplicity of one file; rejected in favor of per-stride atlases.
- Scale-down to 16×16 at import: loses the resolution benefit; rejected.
- Pixel-coordinate manifest (raw px offsets instead of grid indices): harder
  to reason about and inconsistent with existing `x`/`y` convention; rejected.
