# Analysis Tools

Code analysis infrastructure for the BrogueCE TypeScript port. Pre-computes indexes and reference data so that codebase queries are file reads instead of multi-file scans.

## Quick Start

```bash
cd tools/analysis
npm install              # first time only

# Generate fresh stub report + port health (run at start of session)
npx tsx scan-stubs.ts
npx tsx port-health.ts
```

The C manifest is already committed and doesn't need regenerating (the C source is static).

## Components

### C Function Manifest (static)

**Script:** `generate-c-manifest.sh`
**Output:** `docs/analysis/c-manifest.json` (committed)

Pre-computed reference for all 818 C functions in `src/brogue/`. For each function: name, file, line range, return type, parameters, callers, callees, and game system classification. Generated by running 3 CodeQL queries against the `brogue-c` database and merging results via `merge-codeql-results.mjs`.

Never needs recomputing — the C source is frozen.

**Example lookups** (via jq or Node):
```bash
# Who calls attack()?
node -e "const m=JSON.parse(require('fs').readFileSync('docs/analysis/c-manifest.json','utf8')); console.log(m.functions.attack.callers)"

# What does attack() call?
node -e "const m=JSON.parse(require('fs').readFileSync('docs/analysis/c-manifest.json','utf8')); console.log(m.functions.attack.callees)"

# All combat system functions
node -e "const m=JSON.parse(require('fs').readFileSync('docs/analysis/c-manifest.json','utf8')); console.log(m.systems.combat)"
```

### TS Stub Scanner (on-demand)

**Script:** `scan-stubs.ts`
**Output:** `docs/analysis/stub-report.json` (gitignored)

Scans all `.ts` files in `rogue-ts/src/` using the TypeScript compiler API (`ts.createSourceFile`, parse-only). Detects stub arrow functions in object literals: `() => {}`, `() => false`, `() => 0`, `() => null`, `() => true`, `() => []`, `() => ""`, `() => -1`.

Runs in ~2 seconds. Produces:
- Per-stub entries with file, line, property name, stub type, enclosing context builder
- Aggregation by function name (how many context builders stub each function)
- Aggregation by context builder (how complete each builder is)

### Port Health Reporter (on-demand)

**Script:** `port-health.ts`
**Output:** `docs/analysis/port-health.json` + `docs/analysis/PORT_HEALTH.md` (both gitignored)

Cross-references the C manifest with the stub report. Maps stub names to C functions, classifies them by game system, and identifies which stubbed functions are on gameplay-critical paths (reachable from the game loop).

Produces:
- System breakdown: per-system function count, stubbed count, critical count
- Critical stubs: every stub reachable from `playerMoves`, `monstersTurn`, `updateEnvironment`, `startLevel`, etc.
- Most-stubbed context builders ranking
- Human-readable `PORT_HEALTH.md` summary

## File Layout

```
tools/analysis/
  package.json                # typescript + tsx devDependencies
  generate-c-manifest.sh      # One-time: runs CodeQL, produces c-manifest.json
  merge-codeql-results.mjs    # Helper: merges 3 CodeQL query outputs into manifest JSON
  scan-stubs.ts               # On-demand: TS AST stub scanner
  port-health.ts              # On-demand: cross-reference + health report
  CONTEXT.md                  # This file

docs/analysis/
  c-manifest.json             # Committed — static C function reference
  stub-report.json            # Gitignored — regenerated by scan-stubs.ts
  port-health.json            # Gitignored — regenerated by port-health.ts
  PORT_HEALTH.md              # Gitignored — human-readable health summary
```
